# Research Report Orchestrator 项目规则

## 项目目标

Research Report Orchestrator 是一个面向中文行业研究和公司研究的 Codex Skill，用于把 AI 辅助投研工作从一次性生成报告，升级为可复用、可审核、可返工的研究生产流程。

本项目不是算法工程包装，而是一个 AI 辅助投研流程设计与行业研究效率工具。它重点展示：

- 研究任务拆解
- 资料质量控制
- 行业 / 公司分析框架
- 逻辑审核与反方观点检查
- 证据底稿和论点来源映射
- 报告结构化写作
- 券商研报风格打磨
- 可复用多角色工作流
- 适合 GitHub 展示的项目化组织

## 适用场景

适用于：

- 中文行业研究报告
- 中文公司研究报告
- 二级市场投研支持材料
- 实习 / 求职作品集中的研究流程展示
- 将公开资料转化为结构化研究框架
- 对已有研究草稿进行逻辑审核、重写和券商风格打磨
- 需要资料搜集、分析、审核、编辑、QC 分阶段留痕的研究任务

默认边界：

- 地区：中国市场，必要时加入全球对比
- 时间：优先使用最近三年资料，长期趋势可回溯五年
- 语言：中文
- 输出：Markdown / Word 优先，必要时扩展到 PPT / PDF
- 读者：管理者、投资研究人员、求职研究者或需要形成判断的专业读者

## 不适用场景

不适用于：

- 自动生成无来源支撑的投资建议
- 替代持牌投顾、证券分析师或合规审批
- 使用、整理或泄露未授权的内幕信息、客户资料、商业秘密或实习单位保密材料
- 对实时交易作出买卖指令
- 伪造数据、引用、机构观点、财务指标或专家访谈
- 将二手资料包装成一手调研结论
- 为了展示效果牺牲事实准确性或来源可追溯性

如用户请求触及上述边界，应明确改用公开、合规、可追溯资料进行一般性研究分析。

各角色只能修改自己负责的输出文件。主控可以创建任务文件、汇总状态、决定返工和生成最终交付，但不得掩盖前序缺陷。

## 角色边界

- Master Agent：确认任务、维护流程、拆解角色、判断返工和交付状态。
- Researcher：搜集事实、来源、数据口径和冲突，不做最终行业判断。
- Evidence Manager：建立证据底稿、来源索引和论点来源映射。
- Analyst：基于已验证材料形成行业 / 公司逻辑，区分事实、推断、假设和建议。
- Risk & Counterargument Reviewer：输出风险点、反方观点、结论失效条件和后续跟踪指标。
- Logic Reviewer：检查证据链、因果关系、定义口径、时间周期和结论强度。
- Editor：优化结构、语言和可读性，不新增未验证事实。
- Quality Controller：打分、验收或退回，不替前序角色修补。
- Final Report Rewriter：把过程材料转化为完整报告，而不是保留流程记录。
- Sell-side Polisher：强化观点表达、章节标题、图表框架和券商研报风格，不添加无来源结论。

## 质量标准

最终报告至少应包含：

- 执行摘要
- 研究范围与方法
- 核心判断
- 证据与分析
- 风险与反方观点
- 结论或建议
- 限制说明
- 来源清单

质量控制采用 100 分制：

- 准确性：30
- 证据完整性：25
- 逻辑严密性：20
- 可读性：15
- 任务匹配度：10

低于 85 分不得发布。返工必须说明责任角色、具体问题、需要修改的文件和验收条件。最多返工两轮；两轮后仍未通过，应停止自动推进并提交人工决策清单。

## 来源与证据规则

不得虚构事实、数据、引用、机构、人物或来源。

关键事实必须记录：

- 来源名称
- 链接或本地路径
- 发布日期
- 访问日期
- 来源类型
- 可信度
- 数据口径、单位、地域和统计周期

资料优先级：

1. 政府与监管机构
2. 公司公告与财报
3. 行业协会
4. 学术研究
5. 权威数据库
6. 可信媒体
7. 其他二手资料

当资料冲突时，不得擅自选择一个结论。应列出冲突、口径差异和可能原因。

## 保密边界

本项目只应处理公开、授权、可引用的资料。

不得写入仓库或示例文件的内容包括：

- 实习单位内部文件
- 未公开研究材料
- 客户名称、交易信息或项目代号
- 内部会议纪要
- 未公开财务数据
- 任何可能违反 NDA、合规要求或职业伦理的信息

用于 GitHub 展示的 demo 应使用公开资料、模拟数据或明确脱敏后的材料。

## GitHub 展示原则

项目展示应突出研究流程设计和质量控制机制，而不是暗示 AI 能自动替代研究员。

README 应清楚说明：

- 这个 Skill 解决什么问题
- 为什么投研报告不能只靠一次性生成
- 多角色流程如何降低幻觉和逻辑错误
- 证据底稿与论点来源映射如何保留研究过程
- 如何运行一次完整研究任务
- 输出文件长什么样
- 质量控制如何工作
- 项目边界和合规声明

如提供示例，应使用脱敏或公开资料，并明确其仅为 workflow demo。
