# Sell-side Polish Checklist

Use this near the end of a report project, especially for internship, portfolio, or mentor-facing deliverables.

## Final Identity

The final artifact should read as a complete industry/company research report, not as:

- research memo
- evidence audit
- process record
- literature review
- source summary

The report should keep source discipline in the background while foregrounding the industry/company logic.

## Structure

- Does the report answer one central question?
- Does every chapter support the central thesis?
- Is the structure organized by industry/company logic rather than source order?
- Are repeated sections merged?
- Are cases used as support rather than the main spine?

## “Why” Logic

Prefer explanatory headings:

- Weak: `CRO 介绍`
- Strong: `为什么创新药景气会向 CRO 传导？`

- Weak: `支付体系`
- Strong: `为什么支付决定创新药真正赚钱？`

- Weak: `License-out`
- Strong: `为什么 License-out 是开始，不是终点？`

Rewrite encyclopedia-style sections:

- `CRO 负责什么` → `为什么创新药景气会向 CRO 传导？`
- `CDMO 介绍` → `为什么商业化兑现后，CDMO 弹性更偏后周期？`
- `支付体系有哪些` → `为什么支付体系决定创新药真正赚钱？`
- `国际化包括什么` → `为什么海外注册和商业化比交易新闻更重要？`

## Chapter Takeaways

Add a short `本章小结` to serious reports. It should state the point, not summarize every paragraph.

Good pattern:

`本章小结：创新药主线的本质，是行业从“研发投入”进入“价值兑现”。产业链研究的重点，也应从单个产品或交易，转向价值如何在研发、临床、生产、支付和国际化环节中传导。`

## Visuals

Prefer:

- value chain diagram
- commercialization path
- global expansion path
- segment comparison table with short cells
- case box
- chapter conclusion box

For serious reports, create at least one core framework visual near the beginning. It should explain the report’s logic, not decorate it.

Avoid:

- wide tables with long Chinese prose
- dense source audit tables in the main body
- diagrams that merely decorate rather than explain logic

## Language

Make the prose sound like a thoughtful analyst, not a process log.

Do:

- lead with judgment
- explain mechanism
- use concise transitions
- write limitations once, in the right place

Avoid overusing:

- “当前证据强度”
- “不能外推”
- “不构成高置信事实”
- “需要进一步验证”

These belong in methods, risks, limitations, or footnotes unless central to the argument.

## Final Gate

Before delivery, check:

- central thesis is clear within the first page
- each chapter has a reason to exist
- repeated chapters or duplicated themes have been merged
- evidence supports key claims
- risks and limitations are centralized
- cases are supporting boxes rather than the spine
- chapter takeaways are present where useful
- headings explain “why,” not only “what”
- document is editable
- no wide-table layout bug
- no unsupported investment recommendation
